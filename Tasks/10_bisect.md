# Задача 10 — `git bisect`

## Новая команда

```bash
git bisect start           # начать поиск
git bisect bad             # текущий коммит сломан
git bisect good <хэш>      # этот коммит был рабочим
git bisect good / bad      # отвечать на каждом шаге
git bisect reset           # завершить поиск
```

`bisect` реализует бинарный поиск по истории коммитов.
Если что-то сломалось, а вы не знаете когда — `bisect` найдёт виновный коммит
за O(log N) шагов, не перебирая все коммиты вручную.

---

## Физическая модель

Я специально вношу ошибку в репозиторий:
Ваша задача — найти сломанный коммит как детектив, не читая каждый.

Критерий «сломано»: энергия системы за первые 100 шагов вырастает более чем в 10 раз.

---

## Задание

### Шаг 1 — Убедиться что текущая версия сломана

```bash
python main.py  # или скрипт проверки энергии
```

Энергия расходится — что-то не так.

### Шаг 2 — Найти последний рабочий коммит

```bash
git log --oneline
```

Найдите тег `v1.0` — он был поставлен на рабочую версию в задаче 06:

```bash
git show v1.0
```

### Шаг 3 — Запустить бинарный поиск

```bash
git bisect start
git bisect bad                    # текущий HEAD сломан
git bisect good v1.0              # v1.0 был рабочим
```

Git переключит репозиторий на средний коммит. Проверьте:

```bash
python check_energy.py  # ваш скрипт из задачи 06
```

Ответьте Git:
```bash
git bisect good   # если энергия стабильна
git bisect bad    # если энергия расходится
```

Повторяйте до тех пор, пока Git не найдёт коммит с ошибкой.

### Шаг 4 — Изучить виновный коммит

```bash
git show <хэш найденного коммита>
```

Найдите строку с ошибкой. Запишите: что именно было изменено?

### Шаг 5 — Завершить поиск

```bash
git bisect reset  # вернуться на HEAD
```

---

## Подсказки

- Можно автоматизировать: `git bisect run python check_energy.py` — Git сам будет запускать скрипт
- `bisect` работает тем быстрее, чем длиннее история: 1000 коммитов — за 10 шагов
- Если на каком-то коммите код не запускается вообще — `git bisect skip`

---

## Ожидаемый результат

- Виновный коммит найден и изучен
- Вы понимаете принцип бинарного поиска на практике
- Написан скрипт `check_energy.py` который возвращает 0 (ОК) или 1 (ошибка) — это стандартный интерфейс для `bisect run`
