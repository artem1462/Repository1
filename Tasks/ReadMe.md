Артем, предложение доработать репозиторий следующим образом. В данном файле я создаю задачи для изучения git и немного по доработке программы, чтобы научится доводить программы до более читаемых и простых к восприятию.
Задачи идут по нарастающей. Начинать с середины можно, но не рекомендуется — каждая следующая опирается на предыдущую и в коде, и в понимании Git.
Параметры `string()`: `num` - число частиц, длина струны, `k` - коэффицент упругости пружин, `t` - продолжительность симуляции, `math_dt` - время между обсчетами частиц, `anim_dt` - время между кадрами, `anim_speed` - отношение длины симуляции к длине анимации, `anim_show` - выбор способа визуализации частиц, `fixate_edges` - выбор фиксировать ли края струны, `initial_velocity` - начальная скорость возбуждённой частицы, `excited_particle` - индекс возбуждённой частицы.
