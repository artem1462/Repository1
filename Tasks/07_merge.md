# Задача 07 — `git merge`

## Новая команда

```bash
git merge <ветка>            # влить ветку в текущую
git merge --no-ff <ветка>   # влить с явным коммитом слияния
git merge --abort            # отменить слияние при конфликте
```

Слияние объединяет историю двух веток.
`--no-ff` (no fast-forward) создаёт отдельный коммит слияния — это важно:
в истории будет видно, что именно и когда было добавлено в основную ветку.

---

## Физическая модель

В ветке `feature/damping` у вас готов код затухания.
В ветке `feature/initial-conditions` — параметризованные начальные условия.
Пришло время собрать всё вместе в `main` и исследовать оба эффекта одновременно:
как начальные условия влияют на характер затухания?

---

## Задание

### Шаг 1 — Влить первую ветку

```bash
git checkout main
git merge --no-ff feature/initial-conditions -m "merge: параметризованные начальные условия"
```

### Шаг 2 — Влить вторую ветку

```bash
git merge --no-ff feature/damping -m "merge: затухание колебаний"
```

Возможен **конфликт** — если обе ветки изменяли одни и те же строки.

### Шаг 3 — Разрешить конфликт (если возник)

Git пометит конфликтующие места в файле:

<<<<<<< HEAD
vlcs = vlcs + accs * dt
vlcs = vlcs + accs * dt - damping * vlcs * dt
feature/damping

Оставьте нужный вариант (или объедините оба), удалите маркеры, затем:

```bash
git add physics.py
git commit -m "merge: разрешён конфликт в physics.py"
```

## Подсказки

- `git log --oneline --graph --all` после слияния покажет красивый граф с явной точкой слияния
- При конфликте не паникуйте: `git status` всегда покажет какие файлы конфликтуют
- `git merge --abort` отменяет слияние и возвращает всё как было — это безопасно
