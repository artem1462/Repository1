# Задача 03 — `git add -p` и `git commit`

## Новые команды

```bash
git add -p                        # интерактивное добавление по кускам
git commit -m "сообщение"         # зафиксировать изменения
git commit --amend                # исправить последний коммит
git status                        # посмотреть состояние файлов
```

`git add -p` позволяет выбирать, какие именно изменения включить в коммит —
даже если вы изменили несколько несвязанных вещей в одном файле.
Это ключевой навык: хороший коммит содержит одно логическое изменение, а не всё подряд.

---

## Физическая модель

Сейчас в коде с пружной все реализовано в одном файле. Физика, анимация и запуск перемешаны в одном файле.
В этой задаче нужно навести порядок и зафиксировать каждое изменение отдельным коммитом.
Хорошая история коммитов — это документация того, как развивался проект.
Предлагаю старый файл с задачами перемести в папку Archive.

## Задание

Работаем в ветке `feature/initial-conditions` из предыдущей задачи.

### Шаг 1 — Разбить код на модули

Создайте три файла:

**`physics.py`** — всё что касается расчётов:
- `generate_particles()`
- `process_particles()`
- `time_process()`

**`animation.py`** — всё что касается отрисовки:
- `animate_particles()`
- `update()`

**`main.py`** — только вызов:
```python
from physics import *
from animation import *
string(100, 10000, 0.5, 18000, 120, 0.01, 0.04, anim_speed=6)
```

### Шаг 2 — Коммитить по одному изменению

```bash
git add -p
```

Git будет показывать куски изменений и спрашивать: включить в коммит? (`y` — да, `n` — нет, `s` — разбить кусок мельче).

Сделайте **три отдельных коммита**:
```bash
git commit -m "task-03: выделен модуль physics.py"
git commit -m "task-03: выделен модуль animation.py"
git commit -m "task-03: создан main.py как точка входа"
```

### Шаг 3 — Добавить `.gitignore`

Создайте файл `.gitignore`:
```
__pycache__/
*.pyc
.DS_Store
*.png
*.mp4
```

```bash
git add .gitignore
git commit -m "task-03: добавлен .gitignore"
```

## Подсказки

- Если в последнем коммите опечатка в сообщении — исправьте через `git commit --amend`
- `git status` всегда показывает текущее состояние: что изменено, что добавлено в staging, что не отслеживается
- Один коммит = одна идея. Через полгода вы скажете себе спасибо
