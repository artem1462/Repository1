# Задача 08 — Pull Request и code review

## Новая концепция

Pull Request (PR) — это не команда Git, а инструмент GitHub.
Это предложение влить одну ветку в другую, оформленное как обсуждение:
можно оставлять комментарии к конкретным строкам кода, задавать вопросы, просить правки.

Именно так работает рецензирование кода в открытых проектах и научных группах.

---

## Физическая модель

Следующий шаг развития модели — перейти от одномерной струны к двумерной мембране.
Пружины теперь соединяют узлы сетки по четырём направлениям: вверх, вниз, влево, вправо.
Это серьёзное изменение архитектуры — именно такие изменения оформляют через PR,
чтобы тот кто отвечает за проект мог проверить физику и код до того, как они попадут в `main`.

## Задание

### Шаг 1 — Создать ветку для мембраны

```bash
git checkout -b feature/2d-membrane
```

### Шаг 2 — Реализовать мембрану

В `physics.py` добавьте функцию генерации сетки N×N:

```python
def generate_membrane(n, spacing=1.0):
    """Создаёт двумерную сетку N×N частиц"""
    num = n * n
    x = np.linspace(0, spacing * (n-1), n)
    y = np.linspace(0, spacing * (n-1), n)
    xx, yy = np.meshgrid(x, y)
    crds = np.column_stack([xx.ravel(), yy.ravel()])
    vlcs = np.zeros((num, 2))
    accs = np.zeros((num, 2))
    mass = np.ones((num, 2))
    return (crds, vlcs, accs, mass, num)
```

Матрица пружин для сетки: соединить соседей по горизонтали и вертикали.

Добавить удар по центру:
```python
center = (n // 2) * n + n // 2
vlcs[center] = [0, 300]  # скорость в центре мембраны
```

Визуализация через `imshow`: в каждый кадр перекладывать Y-координаты в матрицу N×N.

### Шаг 3 — Открыть Pull Request

1. Запушите ветку:
```bash
git push origin feature/2d-membrane
```
2. На GitHub появится кнопка **Compare & pull request** — нажмите её
3. Напишите описание PR: что реализовано, какая физика, что проверено
4. Отметьте преподавателя как reviewer

### Шаг 4 — Ответить на review

Я могу оставит комментарии к конкретным строкам.
Внесите правки, запушьте — PR обновится автоматически:

```bash
git add .
git commit -m "fix: учтены замечания code review"
git push origin feature/2d-membrane
```

---

## Подсказки

- Хорошее описание PR отвечает на три вопроса: что сделано, зачем, как проверить
- Один PR — одна задача. Не мешайте несвязанные изменения в один PR
- Коммиты внутри PR можно добавлять сколько угодно раз до слияния

## Ожидаемый результат

- Мембрана симулируется и визуализируется
- PR открыт на GitHub с понятным описанием
- После code review замечания учтены
- Преподаватель сливает PR в `main`
