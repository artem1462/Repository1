# Задача 11 — `git revert`

## Новая команда

```bash
git revert <хэш>           # создать коммит отменяющий указанный
git revert HEAD            # отменить последний коммит
git revert --no-commit <хэш>  # применить отмену без коммита
```

`revert` создаёт **новый коммит**, который отменяет изменения из указанного.
История при этом не переписывается — это важно в общем репозитории.
В отличие от `reset`, `revert` безопасен даже если другие уже скачали вашу историю.

---

## Физическая модель

В задаче 10 вы нашли сломанный коммит.
Теперь нужно его отменить — но аккуратно, не трогая коммиты которые шли после него.
Именно для этого нужен `revert`: он «вычитает» конкретное изменение из истории,
оставляя всё остальное нетронутым.

---

## Задание

### Шаг 1 — Убедиться в хэше виновного коммита

```bash
git log --oneline
```

Возьмите хэш коммита найденного в задаче 10.

### Шаг 2 — Отменить виновный коммит

```bash
git revert <хэш>
```

Git откроет редактор с сообщением коммита — можно оставить как есть или написать своё:
    revert: отменён коммит с ошибкой в знаке коэффициента жёсткости

### Шаг 3 — Проверить что симуляция работает

```bash
python check_energy.py
python main.py
```

Энергия должна снова держаться стабильно.

### Шаг 4 — Изучить историю

```bash
git log --oneline
```

Обратите внимание: в истории теперь есть и сломанный коммит, и коммит его отменяющий.
История не переписана — это принципиально.

### Шаг 5 — Сравнить с `git reset`

Для понимания разницы — прочитайте:

```bash
git help reset
git help revert
```

В двух словах:
- `git reset` — переписывает историю (опасно в общем репозитории)
- `git revert` — добавляет коммит отмены (безопасно всегда)

---

## Подсказки

- Если `revert` вызвал конфликт — разрешите его так же как при `merge`, затем `git revert --continue`
- Отменять можно любой коммит из истории, не только последний
- `git revert` можно отменить через ещё один `git revert` — это симметричная операция

---

## Ожидаемый результат

- Симуляция снова работает корректно
- История репозитория содержит и ошибку, и её отмену — это честная летопись проекта
- Вы понимаете когда использовать `revert`, а не `reset`
