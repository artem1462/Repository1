# Задача 01 — `git clone`

## Новая команда

```bash
git clone <url>
```

Клонирование создаёт полную локальную копию репозитория вместе со всей историей коммитов.
После клонирования у вас на машине есть всё то же самое, что хранится на GitHub.

---

## Физическая модель

Вы создали репозиторий через браузер и загрузили туда симуляцию колебаний струны.
Пока что вы редактируете код только через интерфейс GitHub — это не всегда неудобно для реальной работы.
Чтобы запускать, изменять и отлаживать код локально, нужно скачать репозиторий на свой компьютер.

---

## Задание

1. Откройте страницу репозитория на GitHub
2. Нажмите кнопку **Code** → скопируйте HTTPS-ссылку
3. В терминале выполните:

```bash
git clone <скопированная ссылка>
cd <название папки>
```

4. Убедитесь, что код на месте:

```bash
ls
python main.py   # или как называется ваш файл
```

5. Посмотрите, что Git знает о репозитории:

```bash
git remote -v
git log --oneline
```

---

## Подсказки

- `git remote -v` покажет, откуда был клонирован репозиторий — это называется `origin`
- `git log --oneline` выведет список всех коммитов в сжатом виде
- Если появилась ошибка про SSH — используйте HTTPS-ссылку (начинается с `https://`)
- Если не удается разобраться можно писать мне на почту vik-nikol@bk.ru

---

## Ожидаемый результат

- Репозиторий скачан и запускается локально
- Команда `git log --oneline` показывает ваш первый коммит
- Вы понимаете разницу между локальным репозиторием и удалённым (`origin`)
