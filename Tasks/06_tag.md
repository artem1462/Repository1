# Задача 06 — `git tag`

## Новая команда

```bash
git tag v1.0                        # создать лёгкий тег
git tag -a v1.0 -m "описание"       # создать аннотированный тег
git tag                              # список тегов
git push origin --tags              # отправить теги на GitHub
git checkout v1.0                   # переключиться на тег
```

Тег — это именованная метка на конкретном коммите.
В отличие от ветки, тег не двигается: он навсегда закреплён за версией.
В науке это аналог «версии модели, результаты которой опубликованы в статье».

---

## Физическая модель

Прежде чем двигаться дальше — нужно убедиться, что базовая модель работает корректно.
Для этого проверим сохранение энергии: в системе без затухания полная энергия должна оставаться постоянной.

Полная энергия системы пружин:

$$E = \underbrace{\sum_i \frac{m_i v_i^2}{2}}_{\text{кинетическая}} + \underbrace{\sum_{ij} \frac{k_{ij}(r_{ij} - l_{ij})^2}{2}}_{\text{потенциальная}}$$

Если энергия растёт — шаг интегрирования слишком большой и модель численно нестабильна.

## Подсказки

- Посмотреть тег на GitHub можно в разделе **Releases / Tags**
- `git checkout v1.0` переводит репозиторий в состояние «detached HEAD» — это нормально для просмотра, но не для работы
- Для возврата к работе: `git checkout main`
